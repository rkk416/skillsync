# SkillSync Team Development Guidelines

## Branch Strategy

Long-lived branches:

- `main`
- `develop`

Feature branches:

- `feature/auth`
- `feature/profile`
- `feature/placement`
- `feature/collaboration`
- `feature/recommendation`

## Rules

1. No direct pushes to `main`.
2. A Pull Request is required for every merge.
3. Every contributor must make a daily commit while actively working on an assigned module.
4. Do not change the database schema without team leader approval.
5. Do not create duplicate model classes.
6. Do not create duplicate database connection classes.
7. Do not modify service interfaces without approval.
